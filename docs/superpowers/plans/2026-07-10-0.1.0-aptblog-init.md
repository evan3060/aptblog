# aptblog 0.1.0 站点初始化与迁移实施计划

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 完成 aptblog Hexo 站点从零初始化、aptbot learn 18 篇文章迁移、VPS 首次部署、CI/CD 自动部署配置、publish-post skill 建立，最终交付可日常发文的技术博客站点。

**Architecture:** Hexo 静态站点生成器 + Next 主题 + i18n 双语 + GitHub Actions CI/CD + nginx 静态托管。迁移脚本作为一次性工具处理 aptbot learn 文章到 Hexo frontmatter 的格式转换。publish-post skill 封装日常发文流程。

**Tech Stack:** Hexo 7.x、hexo-theme-next、hexo-generator-i18n、hexo-generator-sitemap/feed/search、Giscus、GitHub Actions、rsync、Node.js 20、gray-matter（frontmatter 解析）

## Global Constraints

- VPS 用户 aptblog，SSH 密钥登录已验证通过（见 spec §2.1）
- nginx 站点 blog.aptbot.de 已配置，SSL 证书已签发（见 spec §2.1）
- 源文章 18 篇（非 19 篇，编号跳过 12），中英双语 36 文件，位于 `aptbot/src/learn/articles/`
- 图片引用格式统一为 `/learn/articles/images/<name>.png`（经 grep 验证）
- 禁止引用 GFW 阻断的外部资源（Google Fonts 等）
- dev-workflow 遵循 `docs/superpowers/dev-workflow.md`，task 分 code/content 两类
- plan 不含代码实现，代码在 TDD 阶段根据测试错误驱动编写

## Task 2: 迁移脚本开发（code 类，TDD）

**Files:**
- Create: `scripts/migrate-from-aptbot.mjs`、`tests/migrate-from-aptbot.test.mjs`、`vitest.config.mjs`
- 依赖：Task 1（gray-matter 已安装）

**Interfaces:**
- Consumes: aptbot `src/learn/articles/*.md`（frontmatter 由 zod schema 约束，字段见 spec §3.3）
- Produces: `source/_posts/<slug>.md`（中文）、`source/_posts-en/<slug>.md`（英文）、`source/images/<slug>/*.png`

**行为描述：**
迁移脚本读取 aptbot 源文章，按 spec §3.3 映射规则转换 frontmatter（title/description/tags 直接映射，lastUpdated→date，track+chapter→categories，difficulty/estimatedReadingTime→自定义字段保留，order 丢弃，status 决定输出目录），重写图片引用路径（`/learn/articles/images/<name>.png` → `/images/<slug>/<name>.png`），复制图片到 `source/images/<slug>/`。

脚本支持 CLI 参数 `--source=<path>`（默认 `../aptbot/src/learn/articles/`）和 `--dry-run`（仅打印不写文件）。幂等性：每次运行清空目标目录后重新生成。

**TDD 用例描述（文字）：**
1. 给定一篇 published 中文文章，转换后 frontmatter 含 title/description/tags/date/categories/difficulty/reading_time/prerequisites，不含 order/track/chapter/status/lastUpdated/estimatedReadingTime
2. 给定一篇 published 英文文章（.en.md），输出到 _posts-en/ 而非 _posts/
3. 给定一篇 planned 文章，输出到 _drafts/ 而非 _posts/
4. 图片引用路径 `/learn/articles/images/foo.png` 重写为 `/images/<slug>/foo.png`
5. --dry-run 模式不创建任何文件，仅 stdout 打印操作
6. 二次运行结果一致（幂等性）：先清空目标目录再生成
7. 源目录不存在时报错退出（非 0 退出码）
8. 单篇 frontmatter 解析失败时警告+跳过，继续处理其他文章

**验证命令：**
- `npx vitest run tests/migrate-from-aptbot.test.mjs` 全部通过
