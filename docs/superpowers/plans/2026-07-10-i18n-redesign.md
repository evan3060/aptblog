# aptblog i18n 全局语言切换重设计 实施计划

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 实现 aptblog 全局语言切换功能，修复中英文混合显示问题，使中文站点全中文、英文站点全英文，顶部导航栏有 aptbot 风格的 pill 语言切换按钮。

**Architecture:** 路径前缀方案（`/` 中文 + `/en/` 英文），自定义 Hexo 生成器生成分离的中英文首页/归档/分类/标签页，主题原生 i18n 语言文件翻译 UI 文案，aptbot 风格 pill 按钮注入顶部导航栏通过 localStorage 持久化语言选择。

**Tech Stack:** Hexo 7.x、hexo-theme-next 8.28.0、vitest、gray-matter、nunjucks 模板

**Spec:** `docs/superpowers/specs/2026-07-10-i18n-redesign-design.md`

## Global Constraints

- 语言代码：`zh-CN`（非 `zh`），匹配主题 `zh-CN.yml` 语言文件
- 默认语言（zh-CN）URL 无前缀，英文 URL 加 `/en/` 前缀
- 语言切换器样式：pill 胶囊按钮，`border-radius: 9999px`，视觉风格与 aptbot 一致
- 持久化：`localStorage['aptblog.lang']`，不以 localStorage 触发自动跳转
- 文章 lang 字段：中文为 `zh-CN`，英文为 `en`
- 迁移脚本中文文章无 permalink 字段，英文文章 `permalink: en/:slug.html`

### Task 4: 语言切换器 UI 和交互

**Files:**
- Create: `source/_data/header.njk`
- Create: `source/_data/styles.styl`
- Create: `source/_data/body-end.njk`

**Interfaces:**
- Consumes: Task 1 的 `custom_file_path` 配置
- Consumes: Task 3 的 `page.lang` 变量（由主题 template_locals 设置）
- Produces: 顶部导航栏的 pill 语言切换按钮
- Produces: localStorage 持久化和首次访问语言探测逻辑

**描述：**
创建三个自定义文件注入到主题：

`source/_data/header.njk`：在顶部导航栏 `.main-menu` 末尾注入一个 `<li>` 元素，含 pill 按钮。按钮 id 为 `lang-switcher`，根据 `page.lang` 显示「EN」（当前中文时）或「中」（当前英文时）。

`source/_data/styles.styl`：pill 按钮样式，模仿 aptbot：`border-radius: 9999px`、`border: 1px solid`、`padding: 6px 14px`、`font-size: 14px`、`cursor: pointer`。移动端响应式：`font-size: 12px`、`padding: 4px 10px`。

`source/_data/body-end.njk`：JS 逻辑：
1. 读取 `document.documentElement.lang` 判断当前语言
2. 设置切换按钮显示文字
3. 点击事件：计算目标 URL（`buildEnPath` 或 `buildZhPath` 逻辑的客户端版本），写入 `localStorage['aptblog.lang']`，跳转
4. 首次访问探测：`localStorage['aptblog.lang']` 未设置时，检查 `navigator.language`，非 zh 开头且当前不在 `/en/` 则跳转到 `/en/`，然后写入 localStorage

- [ ] **Step 2: 创建 header.njk**
